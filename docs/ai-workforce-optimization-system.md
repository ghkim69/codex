# AI 기반 인력운영 최적화 시스템 설계서

## 1. 목표
MES의 실시간 생산정보, 작업실적, 공정별 작업부하, 작업자 숙련도, 근무 가용정보를 통합 분석하여 생산공정별 최적 인력배치안을 자동 산출하는 시스템을 설계한다. 본 시스템의 핵심 목표는 다음과 같다.

- **생산성 향상**: 목표 생산량 대비 실제 산출량을 높이고 유휴시간을 줄인다.
- **공정 병목 최소화**: WIP 증가, 설비 대기, 작업 지연이 집중되는 병목공정을 조기에 완화한다.
- **현장 운영 효율화**: 반장·관리자의 수기 스케줄링 부담을 줄이고 대응 속도를 높인다.
- **설명 가능한 의사결정**: 왜 해당 작업자와 공정이 매칭되었는지 근거를 제공한다.

## 2. 해결하려는 현장 문제
기존 제조 현장은 다음과 같은 문제를 반복적으로 겪는다.

1. **라인별 인력 편차**
   - 특정 공정에 숙련자가 집중되고 타 공정은 작업 지연이 발생한다.
2. **실시간 상황 반영 한계**
   - 결근, 긴급오더, 설비 이상, 불량 급증 등 돌발상황을 수기 계획이 즉시 반영하지 못한다.
3. **숙련도 활용 비효율**
   - 작업자별 다기능 숙련 수준이 체계적으로 반영되지 않아 최적 배치가 어렵다.
4. **공정 부하 예측 부재**
   - 작업량 급증이 예상되는 공정을 사전 식별하지 못해 병목이 뒤늦게 발생한다.
5. **운영 의사결정 표준 부족**
   - 관리자 경험 의존도가 높아 근무조별 편차가 크다.

## 3. 시스템 범위

### 3.1 입력 데이터
- **MES 실시간 생산정보**: 설비 상태, 작업지시, 실적수량, 불량수량, takt time, cycle time
- **작업실적 데이터**: 작업자별 생산량, 품질실적, 작업시간, 공정 투입 이력
- **공정별 작업부하 정보**: 계획/실적 대비 잔여 물량, 병목지수, 공정 대기열, 우선순위 오더
- **작업자 숙련도 정보**: 공정별 레벨, 인증 보유 현황, 교육 이수, 작업 품질 수준
- **근무 가용 정보**: 출결, 교대조, 잔업 가능 여부, 휴게시간, 법정 근로 제약

### 3.2 출력 결과
- 공정별 권장 배치 인원 수
- 작업자별 추천 공정 및 투입 시간대
- 병목 예상 공정 및 대응 우선순위
- 생산성/납기/품질 기준을 반영한 배치 시뮬레이션 결과
- 관리자 승인용 설명 리포트 및 대안 시나리오

## 4. 사용자 역할

| 역할 | 주요 목적 | 주요 기능 |
| --- | --- | --- |
| 생산관리자 | 당일/교대 인력 운영 계획 수립 | 최적 배치안 조회, 시나리오 비교, 승인/수정 |
| 현장반장 | 실시간 재배치 대응 | 결원 발생 시 대체 추천, 병목 경보 확인 |
| 인사/교육 담당 | 숙련도 관리 | 교육 필요 인원 식별, 다기능 인력 육성 계획 |
| 경영진 | 운영 KPI 모니터링 | 생산성 향상률, 병목 감소율, 인력 효율 분석 |

## 5. 핵심 기능

### 5.1 데이터 통합 및 정합성 관리
- MES, ERP, HR, 근태시스템 데이터를 배치 + 스트리밍 방식으로 수집
- 작업자, 공정, 설비, 오더에 대한 마스터 키를 통합
- 결측치, 지연 수신, 이상치 발생 시 보정 규칙 적용

### 5.2 공정 부하 예측
- 최근 실적, 주문 잔량, 설비 가동률, 불량률을 반영해 향후 1~8시간 공정별 부하 예측
- 긴급오더와 계획변경을 이벤트 기반으로 즉시 재반영
- 예측값과 실제값 오차를 지속 학습하여 모델을 개선

### 5.3 작업자 역량 스코어링
- 공정별 숙련도, 최근 생산성, 품질 안정성, 전환적응 속도를 반영한 역량 점수 산정
- 단일 공정 숙련뿐 아니라 멀티스킬 활용도까지 반영
- 일정 기간 이상 미투입 공정은 숙련도 감가 규칙 적용

### 5.4 최적 인력배치 엔진
- 생산량 최대화, 병목 최소화, 초과근무 제약, 필수 자격조건 충족을 동시에 고려
- 수리최적화와 AI 예측 결과를 결합한 하이브리드 방식 적용
- 결과값으로 최적안 1개와 차선안 2~3개를 함께 제시

### 5.5 실시간 재배치 추천
- 결근, 설비고장, 품질이상, 주문 우선순위 변경 이벤트 수신 시 재계산
- 변경 전/후 KPI 영향을 즉시 비교
- 관리자 승인 후 현장 단말에 배치 변경사항 전파

### 5.6 설명 가능한 AI 대시보드
- 추천 근거: 공정부하, 숙련도, 가용시간, 이동비용, 품질 리스크
- 예측 신뢰도, 제약조건 충족 여부, 예상 효과를 시각화
- KPI 변화와 실제 적용 결과를 히스토리로 추적

## 8. 데이터 모델 개요

### 8.1 주요 엔터티
- `Worker`
  - worker_id, 소속, 근무조, 자격, 최대 근무시간, 선호/제한 공정
- `SkillMatrix`
  - worker_id, process_id, skill_level, certification_flag, last_trained_at
- `ProcessLoad`
  - process_id, timeslot, planned_qty, remaining_qty, queue_length, bottleneck_score
- `AttendanceAvailability`
  - worker_id, shift_date, available_slots, overtime_allowed, leave_flag
- `ProductionPerformance`
  - worker_id, process_id, output_per_hour, defect_rate, setup_loss
- `AssignmentPlan`
  - plan_id, timeslot, process_id, assigned_worker_id, expected_output, confidence

### 8.2 데이터 품질 규칙
- 작업자 ID, 공정 ID, 시간 슬롯은 기준정보와 100% 매핑되어야 한다.
- 실적 입력 지연이 5분을 초과하면 배치 결과 신뢰도 경고를 생성한다.
- 숙련도 갱신일이 180일 이상 경과하면 검증 필요 상태로 표시한다.

## 9. AI/최적화 로직 설계

### 9.1 예측 모델
1. **공정 부하 예측 모델**
   - 입력: 오더 잔량, takt/cycle time, 불량률, 설비 가동률, 과거 시간대 패턴
   - 후보 모델: XGBoost, LightGBM, Temporal Fusion Transformer
   - 출력: 시간대별 공정 처리수요, 병목 확률

2. **작업자 생산성 추정 모델**
   - 입력: 숙련도, 최근 작업이력, 품목 난이도, 설비 상태, 교대 시간
   - 출력: 작업자-공정 조합별 기대 생산량 및 품질 리스크

3. **결근/가용성 리스크 모델**
   - 입력: 출결 패턴, 교대정보, 잔업 빈도, 휴무 패턴
   - 출력: 단기 결원 확률, 대체 필요도

### 9.2 최적화 목적함수
다음 목적을 가중합 방식으로 동시에 최적화한다.

- 총 기대 생산량 최대화
- 병목공정 대기시간 최소화
- 숙련도 미달 배치 페널티 최소화
- 작업자 이동/전환 비용 최소화
- 초과근무 및 법규 위반 페널티 최소화
- 품질 리스크 최소화

예시 목적함수:

```text
Maximize
Σ(배치인원 × 기대생산량)
- α·병목대기패널티
- β·숙련도미달패널티
- γ·이동비용
- δ·초과근무패널티
- ε·품질리스크
```

### 9.3 주요 제약조건
- 한 작업자는 동일 시간 슬롯에서 하나의 공정에만 배치 가능
- 특정 공정은 인증 또는 최소 숙련 레벨 이상만 배치 가능
- 교대/휴게/최대 근무시간 준수
- 핵심 공정 최소 필요 인원 보장
- 설비 정지 공정에는 배치 금지 또는 제한 배치
- 노사협약/안전규정 관련 제약 반영

### 9.4 의사결정 방식
- **T-1 계획 최적화**: 교대 시작 전 기본 배치안 계산
- **실시간 재최적화**: 이벤트 발생 시 영향 범위 공정만 부분 재계산
- **승인 기반 실행**: 자동 추천 후 관리자 승인 시 확정 반영

## 10. 운영 화면 설계

### 10.1 종합 상황판
- 공정별 현재 인원, 목표 대비 생산량, 병목위험도 표시
- 경고 우선순위 Top N 공정 표시
- 교대별 인력 충원 필요도 색상 강조

### 10.2 최적 배치안 비교 화면
- 현재 배치 vs AI 추천 배치 vs 관리자 수동안 비교
- 예상 생산량, 병목 해소율, 초과근무 증가 여부 제공
- 변경 시 영향받는 작업자 목록과 사유 표시

### 10.3 작업자 상세 카드
- 멀티스킬 맵, 최근 실적 추이, 품질 지표, 피로도/근무시간 확인
- 대체 가능 공정 및 추가 교육 필요 공정 추천

## 11. 시스템 구현 로드맵

### 단계 1. 진단 및 데이터 정합화 (1~2개월)
- 인터페이스 정의 및 마스터 정비
- 현행 배치 프로세스 분석
- KPI 기준선 수립

### 단계 2. 예측 모델 개발 (2~3개월)
- 공정 부하 예측 및 작업자 생산성 추정 모델 학습
- 데이터 품질 모니터링 구성
- 파일럿 라인 대상 모델 검증

### 단계 3. 최적화 엔진 구축 (2개월)
- 제약조건 모델링
- 배치 시뮬레이터와 What-if 분석 기능 구현
- 운영자 설명 로직 설계

### 단계 4. 현장 적용 및 고도화 (2~3개월)
- 대시보드/알림/승인 워크플로우 구현
- 실시간 재배치 API 적용
- 사용자 피드백 기반 가중치 튜닝

## 13. 보안 및 거버넌스
- 개인정보 및 인사정보는 역할 기반 접근통제(RBAC)로 제한한다.
- 모델 학습/추천 결과에 대한 변경 이력과 승인 이력을 보관한다.
- 추천 결과가 노동법, 안전규정, 내부 운영정책을 위반하지 않도록 정책 엔진을 둔다.
- 모델 편향 검증 기준을 수립하여 특정 작업자군에 불리한 배치가 지속되지 않도록 한다.

## 14. 기대 효과
- 생산계획 변동과 현장 돌발상황에 대한 대응속도를 높일 수 있다.
- 병목공정 완화와 멀티스킬 활용 증대로 생산성 향상을 기대할 수 있다.
- 관리자 경험 중심 운영을 데이터 기반 운영으로 전환할 수 있다.
- 교육 우선순위와 인력 육성 방향을 정량적으로 도출할 수 있다.

## 15. PoC 성공 기준
- 파일럿 라인에서 기존 수기 배치 대비 생산성 5% 이상 향상
- 병목공정 평균 대기시간 10% 이상 감소
- 관리자 배치 수립 시간 30분 이상 절감
- 추천안 수용률 70% 이상 확보
- 실시간 재배치 응답시간 3분 이내 달성

## 16. 향후 확장 방향
